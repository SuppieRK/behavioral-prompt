# Security Policy

## Supported Version

Security fixes are handled on the current `main` branch.

## Reporting a Vulnerability

Do not report security issues in public issues or pull requests.

Use GitHub private vulnerability reporting for this repository. After the
repository is published, private vulnerability reporting must be enabled in the
repository security settings.

Include enough detail to reproduce the issue, such as affected files, commands,
logs, and the expected impact. Do not include secrets or private credentials in
the report.
